# Glenn의 기술용어 위키 (Tech Glossary for VC)
> 문과 출신 벤처캐피털리스트를 위한 반도체·소재·배터리·AI인프라 기술용어 사전
> 개념의 상하관계(계층구조) 기준으로 정리

---

## 계층 구조 맵

```
【소재·화학】
화학 물질
└── 분자 (Molecule)
    ├── 작용기 (Functional Group) ← 분자 물성 결정 원자단
    ├── 단량체 / 모노머 (Monomer) ← 폴리머 구성 단위
    └── 폴리머 (Polymer) ← 모노머 반복 결합 거대분자
        └── 폴리이미드 (PI) ← 고내열, FPCB·패키징

물질 전도성 스펙트럼
전도체 ── 반도체 ── 절연체
└── CNT ← 탄소 육각형 원통, 고전도·고강도
    ├── TIM 소재 (열계면)
    └── 도전재 (배터리 전극 첨가)

화합물 반도체 (다원소 화합)
├── GaN·SiC → 전력반도체
└── GaAs·InP → 광·RF 반도체

【반도체 공정】
전공정 (Front-End)
├── 웨이퍼 제조 → 산화 → 포토리소그래피 → 식각
└── 증착/이온주입 → 금속배선 → (EDS)
    └── 트랜지스터 ← 공정으로 만들어지는 기본 소자
        └── MOSFET ← 트랜지스터의 한 종류 (전압 제어)
    └── 다이오드 ← 또 다른 기본 소자 (한방향 전류)

후공정 (Back-End)
├── OSAT ← 패키징+테스트 전문 외주기업
├── 패키징 연결 기술
│   ├── Wire Bonding (1세대, 범용·저가)
│   ├── BGA/Flip Chip (2세대, 고성능)
│   └── TSV (3D 적층 핵심) → MR-MUF (충전·접합 공정)
│       └── HBM = DRAM TSV 적층
├── 패키지 기판
│   ├── CCL (원재료: 구리+절연판)
│   │   ↓
│   └── MLB (다층 적층 완제품) → ABF 기판(AI칩용 고성능)
└── 테스트
    ├── 프로브카드 ← 웨이퍼 단계 (EDS)
    ├── 핸들러 ← 패키지 칩 이송·분류 장비
    └── 테스트 소켓/핀 ← 패키지 칩 접촉 부품

가공·연결 공정
├── 다이싱 ← 웨이퍼→개별칩 절단
├── 레이저 어닐링 ← 결정구조 복구, 하이브리드본딩 활성화
└── 하이브리드 본딩 ← 솔더볼 없는 차세대 접합

설계 생태계
├── 팹리스 ← 반도체 IP(ARM 등) 활용해 설계
├── 디자인하우스 ← 설계를 파운드리 공정에 맞게 변환
└── 파운드리(TSMC 등) ← 실제 제조

【메모리 제품】
DRAM 계열
├── DDR → DIMM(서버·데스크탑, 133mm) / SO-DIMM(노트북, 67mm)
└── LPDDR → 납땜 직접 부착(스마트폰) / SOCAMM(AI PC, ~40mm)
    └── HBM = LPDDR 고대역폭 버전 (TSV 적층)

NAND 플래시 계열 (비휘발성 저장)
└── MCP/eMCP ← NAND 적층 또는 NAND+LPDDR 통합 패키지
    └── 중저가폰·IoT 주력 제품

【전력·전자부품】
전기의 형태: AC ↔ DC
    └── 전력반도체가 변환 담당
반도체 신호 분류
└── 아날로그 반도체 (연속신호)
    └── 전력반도체 ⊂ 아날로그 반도체
        └── 다이오드, MOSFET, IGBT, GaN·SiC
MLCC ← 전압 안정화 수동부품

【광통신·AI 인프라】
다이오드 → 레이저 다이오드
    ↓
실리콘 포토닉스 (광회로, CMOS 공정)
    ↓
CPO (스위치칩+광엔진 동일 패키지)
    ↓
AI 데이터센터 전력·속도 문제 해결

SiPh 밸류체인: InP웨이퍼 → 레이저 → SiPh파운드리 → DSP → 모듈조립 → 스위치 → 광엔진

【수소·에너지】
연료전지
├── PEMFC (저온, 고분자막, 수소차)
└── SOFC (고온, 세라믹, 발전소·데이터센터)

【배터리·전기차】
배터리 내부 (화학 에너지 저장)
├── 양극재: 에너지밀도 결정 (NCM·NCA·LFP)
├── 음극재: 충방전 속도·수명 결정 (흑연·실리콘)
└── 버스바: 셀 간 대전류 연결

배터리 외부 (전기 에너지 변환·전달)
├── 컨버터: 고전압DC → 저전압DC (보조전원)
└── 인버터: DC → AC (모터 구동, IGBT/SiC 핵심)

【IT 시스템】
ERP (경영 전반 계획·관리) ↔ MES (현장 실시간 모니터링) ↔ SCADA (인프라 원격감시)

데이터베이스
├── 관계형 DB / NoSQL (정형·비정형)
└── 벡터 DB (의미기반 검색) → RAG 핵심 인프라

지식 표현 구조 (벡터DB의 형제 개념)
└── 지식 그래프 (Knowledge Graph) ← 개체(노드)-관계(엣지)로 표현된 데이터
    └── 온톨로지 (Ontology) ← KG가 따르는 스키마/규칙 (설계도 vs 실제 데이터)

【에너지·발전】
터빈 (유체 에너지 → 회전력)
├── 증기터빈 (석탄·원자력 발전소)
├── 가스터빈 (천연가스·수소 혼소, AI 데이터센터 전력)
└── SOFC + 터빈 복합 (초고효율 발전)

【네트워킹·AI인프라】
네트워크 스위치 ← GPU 클러스터 간 초고속 통신
└── CPO = 스위치칩 + 광엔진 통합

【로보틱스·자동화】
액추에이터 ← 전기신호 → 물리적 움직임 ("근육")
├── 모터 액추에이터 (회전운동)
└── 리니어 액추에이터 (직선운동)

【소프트웨어·플랫폼】
API ← 소프트웨어 간 통신 규약
└── AI API (LLM 기능 외부 제공) → 사용량 기반 반복수익

【무선통신·네트워크】
VC 통신기술 분류 3축: ①거리 / ②토폴로지(연결구조) / ③트레이드오프(전력·속도·거리)

거리별 무선통신 분류
├── PAN (~10m): 블루투스, UWB
├── LAN (~100m): Wi-Fi, Zigbee
├── WAN (~수km): LoRa, NB-IoT
└── WWAN (수십~수백km): 5G/LTE, 위성

토폴로지 (연결구조)
├── Star ← AP 중심 (Wi-Fi)
├── Mesh ← 분산·우회 (Zigbee·Matter)
├── P2P ← 1:1 직접 (블루투스 Classic)
└── Ring ← 순환 (산업 이더넷)

유선 네트워크
└── 이더넷 (Ethernet) ← 유선 네트워크 표준
    └── TSN ← 실시간 보장 확장 규격 (스마트팩토리)

AI 클러스터 인터커넥트
├── InfiniBand ← 서버 간 초고속 (NVIDIA·Mellanox)
├── NVLink ← GPU 간 전용 연결 (NVIDIA 독점 생태계)
└── RoCE ← 이더넷 기반 대안 (InfiniBand 경쟁)

프로토콜 스택 / OSI 7계층 ← 통신 계층 구조 이해 틀
```

## 항목별 상세 설명

### 1. 폴리머 (Polymer) 〔상위 개념〕

- **정의**: 작은 분자(단량체/모노머)가 수백~수천 개 반복 결합해 만들어진 거대 분자
- **어원**: poly(많다) + mer(단위)
- **분류**: 합성(플라스틱, 나일론, 실리콘, 에폭시) / 천연(면, 단백질, DNA)
- **반도체 IR 맥락**: 포토레지스트, 식각 부산물, 패키징 봉지재(EMC), 저유전율 절연층
- **IR 팁**: "폴리머 소재"라고 뭉뚱그리는 이유 = 보안 + 복합소재 특성 + 시장정의 편의 + 피벗 유연성

---

### 2. 단량체 / 모노머 (Monomer) 〔폴리머의 구성 단위〕

- **정의**: 폴리머를 구성하는 반복 단위 분자
- **예**: 에틸렌(→폴리에틸렌), 이미드(→폴리이미드), 아미드(→나일론)

---

### 3. 작용기 (Functional Group) 〔분자 분류 기준〕

- **정의**: 분자의 화학적 성격(반응성, 물성)을 결정하는 특징적 원자단
- **핵심 원리**: 분자는 크기가 아니라 작용기의 유무/종류로 분류됨
- **비유**: 혈액형 — 크기가 달라도 혈액형이 같으면 같은 방식으로 반응
- **주요 작용기별 물성**

| 작용기 | 계열 | 핵심 물성 | 용도 |
|--------|------|----------|------|
| 이미드 | 폴리이미드 | 고내열, 고강도 | 패키징, FPCB |
| 아미드 | 나일론 | 기계적 강도 | 범용 구조재 |
| 에틸렌 | 폴리에틸렌 | 범용, 저가 | 일반 플라스틱 |
| 플루오로 | 불소계 폴리머 | 비활성, 발수성 | 고순도 공정 소재 |

---

### 4. 폴리이미드 (Polyimide, PI) 〔폴리머의 하위 종류〕

- **정의**: 이미드(imide) 작용기를 가진 단량체가 반복 결합한 폴리머
- **특성**: 고내열성, 고강도
- **용도**: 반도체 패키징, FPCB(연성회로기판), 디스플레이 기판
- **관련 기업**: SKC, PI첨단소재

---

### 5. 전도체 (Conductor) 〔물질 분류 - 상위 개념〕

- **정의**: 전기(또는 열)를 잘 흘려보내는 물질. 자유전자가 많아 전자 이동 용이
- **스펙트럼**: 전도체(구리·금) ── 반도체(실리콘) ── 절연체(유리·고무)
- **IR 맥락**: 반도체 배선재료(구리, 텅스텐), 전극 소재 선택이 핵심 원가·성능 변수

---

### 6. CNT (Carbon Nanotube, 탄소나노튜브) 〔전도체의 하위 소재〕

- **정의**: 탄소 원자가 육각형 벌집 구조로 연결되어 원통형으로 말린 나노 크기 튜브
- **핵심 특성**: 전기전도성(구리 이상), 열전도성(다이아몬드급), 인장강도(철강의 100배), 무게(철강의 1/6)
- **반도체·소재 IR 맥락**: 방열소재(TIM), 차세대 배선(구리 대체), 복합소재 첨가제, 배터리 도전재
- **원리**: 탄소 전자 4개 중 3개만 결합에 사용, 1개가 자유전자로 풀려 전도성 발생
- **IR 팁**: CNT 단독보다 "CNT 복합소재" 형태가 많음. 매트릭스(폴리머/금속)가 핵심 차별점

---

### 7. TIM (Thermal Interface Material, 열계면소재) 〔냉각 소재〕

- **정의**: 칩과 히트싱크 사이의 미세 공기층을 채워 열전도를 높이는 소재
- **주요 종류**: 써멀그리스(범용), 인듐포일(고성능), CNT기반(차세대), 솔더(최고열전도)
- **IR 맥락**: AI 칩 발열 심화 → TIM 스펙 요구 급상승

---

### 8. 도전재 (Conductive Agent) 〔배터리·소재 공정 용어〕

- **정의**: 전도성을 부여하기 위해 소량 첨가하는 소재. 전도체=개념, 도전재=역할
- **배터리 전극 구조**: 활물질(90%+) + 도전재(1~5%) + 바인더
- **CNT 도전재 주목 이유**: 카본블랙 대비 소량으로 동일 전도성 → 활물질 비중 증가 → 에너지밀도 향상
- **관련 기업**: LG화학, 금호석유

---

### 9. 트랜지스터 (Transistor) 〔반도체 기본 소자〕

- **정의**: 전기 신호를 증폭하거나 스위치(on/off)하는 반도체의 가장 기본 소자
- **디지털 연산 연결**: ON→1, OFF→0
- **구조**: 게이트(수도꼭지) → 소스~드레인 전류 흐름 제어
- **크기와 성능**: 작을수록 집적도↑, 속도↑, 소비전력↓
- **IR 맥락**: 미세화 한계 → 구조 전환(FinFET→GAA)마다 새로운 소재·장비 수요 발생

---

### 10. 프로브카드 (Probe Card) 〔반도체 검사 장비 소모품〕

- **정의**: 웨이퍼 상태의 칩이 정상 작동하는지 검사하기 위해 전기 신호를 주고받는 접촉 장치
- **공정 위치**: 웨이퍼 제조 완료 → [웨이퍼 테스트=EDS] → 불량 선별 → 패키징
- **기술 난이도**: 수십 μm 간격 전극에 수천 핀 동시 균일 접촉 + 수십만 회 반복 정밀도
- **관련 기업**: LEENO(리노공업·국내), FormFactor·Technoprobe(해외)

---

### 11. 테스트 소켓 / 테스트 핀 (Test Socket / Test Pin) 〔패키지 테스트 접촉 부품〕

- **테스트 소켓**: 패키징 완료된 개별 칩을 끼워 넣어 테스트하는 탈착식 접촉 장치
- **테스트 핀**: 소켓 또는 프로브카드 내부의 접점 부품 (독립 장비가 아닌 구성 부품)
- **프로브카드와 차이**: 프로브카드=웨이퍼 단계 / 테스트 소켓=패키징 후 단계
- **관련 기업**: LEENO(리노공업), ISC

---

### 12. OSAT (Outsourced Semiconductor Assembly and Test) 〔반도체 후공정 전문기업〕

- **정의**: 반도체 패키징 + 테스트를 전문으로 외주받아 수행하는 기업/산업
- **공급망 위치**: 설계(Fabless) → 전공정(Foundry) → 후공정(OSAT)
- **중요성 변화**: 과거 단순 보호 역할 → 현재 HBM·2.5D/3D 패키징 등 성능 결정 핵심 기술
- **관련 기업**: ASE(글로벌 1위·대만), Amkor(2위), 네패스·하나마이크론·두산테스나(국내)

---

### 13. 핸들러 (Handler) 〔반도체 테스트 자동화 장비〕

- **정의**: 테스트 과정에서 칩을 자동으로 집어 소켓에 넣고, 테스트 후 양품/불량으로 분류하는 장비
- **테스터와의 관계**: 테스터(두뇌, 전기검사) + 핸들러(손발, 이송·분류) 항상 세트
- **핵심 기술**: 온도 제어(-40℃~150℃), UPH(시간당 처리량), 정밀도
  - 극한온도 핸들링이 어려운 이유: 부품별 열팽창계수(CTE) 차이로 접촉 위치 어긋남, 소재 물성 변화(경화·연화), 온도 균일 유지 어려움
  - 해법: 온도별 위치 보정 알고리즘, CTE 매칭 소재, 탄성 설계(포고핀), 온도 안정화 후 핸들링
- **관련 기업**: 한미반도체(국내), Advantest(일본)

---

### 14. LPDDR (Low Power Double Data Rate) 〔모바일용 DRAM 규격〕

- **정의**: 모바일·저전력 기기용으로 설계된 DRAM 규격
- **DDR과 관계**: DDR(PC·서버용) vs LPDDR(모바일·엣지용)
- **저전력 의미**: DDR5(1.1V) 대비 LPDDR5(0.5~1.05V), 전력 약 50% 절감
- **탑재 방식**: 칩에 직접 납땜 → 속도↑, 전력↓, 단 교체 불가
- **최근 흐름**: 온디바이스 AI 확산 → LPDDR5X 수요 급증. 애플 M시리즈, 퀄컴 스냅드래곤

---

### 15. DIMM (Dual Inline Memory Module) 〔표준 메모리 모듈〕

- **정의**: 메모리 칩 여러 개를 기다란 기판 위에 나란히 붙인 표준 메모리 모듈
- **용도**: 데스크탑 PC, 서버 / 크기 133mm
- **규격 세대**: DDR3 → DDR4 → DDR5
- **특징**: 슬롯에 꽂아 탈착·업그레이드 가능

---

### 16. SO-DIMM (Small Outline DIMM) 〔노트북용 소형 메모리 모듈〕

- **정의**: DIMM을 노트북에 맞게 절반 크기(67mm)로 줄인 모듈
- **기반**: DDR 규격 (LPDDR 아님) → 전력 소비 상대적으로 높음
- **한계**: AI PC 전환 시 저전력 요구 충족 어려움 → SOCAMM으로 대체 흐름

---

### 17. SOCAMM (Small Outline Compression Attached Memory Module) 〔AI PC용 차세대 메모리 폼팩터〕

- **정의**: LPDDR 메모리를 탈착 가능한 모듈 형태로 만든 차세대 폼팩터
- **등장 배경**: LPDDR 저전력 장점 유지 + 납땜 방식의 업그레이드 불가 단점 해소
- **크기**: ~40mm (SO-DIMM 67mm보다 작음)
- **IR 맥락**: MS Copilot+ PC 규격 채택. 삼성·SK하이닉스 LPDDR5X 기반 모듈 경쟁

---

### 18. Wire Bonding (와이어 본딩) 〔패키징 연결 기술 - 1세대〕

- **정의**: 칩 전극과 기판을 금속 와이어(금선·동선)로 연결하는 전통적 패키징 방식
- **장점**: 공정 단순·저가, 오랜 검증 이력, 수리 용이
- **단점**: 와이어 길이만큼 신호 지연, 고속 신호 불리
- **현재 용도**: 저가·범용 칩에 여전히 광범위 사용

---

### 19. BGA (Ball Grid Array) 〔패키징 연결 기술 - 2세대〕

- **정의**: 칩 하단에 솔더볼을 격자 배열해 기판과 연결하는 방식. Flip Chip의 대표 구현
- **Wire Bonding과 차이**: 칩을 뒤집어(Flip) 하단 볼로 직접 접합 → 연결 거리 단축
- **장점**: 고속 신호 유리, 핀 수 대폭 증가, 방열 유리
- **현재 용도**: CPU·GPU·AP 등 고성능 칩 주류 방식

---

### 20. TSV (Through Silicon Via, 실리콘 관통 전극) 〔3D 적층 핵심 기술〕

- **정의**: 실리콘 칩을 수직으로 뚫어 금속을 채워 위아래 칩을 직접 연결하는 기술
- **핵심 장점**: 연결 거리 수mm → 수십μm 단축 → 신호속도↑, 전력↓, 대역폭 폭발적 증가
- **HBM과의 연결**: DRAM 8~12개 수직 적층 + TSV 연결 = HBM. AI칩(H100·B200) 핵심 메모리 구조

---

### 21. MR-MUF (Mass Reflow - Molded Underfill) 〔HBM 패키징 공정 기술〕

- **정의**: TSV 적층 칩 사이 빈 공간을 EMC 수지로 채우면서 동시에 열로 접합하는 공정
- **필요 이유**: TSV 적층 후 칩 사이 공간 방치 시 열팽창 균열·수분 침투·충격 취약
- **공정 순서**: 칩 적층 → Mass Reflow(솔더볼 동시 용융) → MUF(EMC 주입·경화)
- **기존 TC-NCP 대비**: 칩 한 장씩 순차 압착 → 전체 동시 리플로우로 생산성·균일도↑
- **IR 맥락**: 한미반도체 핵심 장비(TC본더) 영역. SK하이닉스 HBM3E 양산 공정 채택

---

### 22. NAND 플래시 (NAND Flash) 〔비휘발성 저장 메모리〕

- **정의**: 전원이 꺼져도 데이터가 유지되는 저장용 메모리
- **DRAM과 비교**: DRAM=임시 작업공간(RAM) / NAND=영구 저장공간(Storage)
- **용도**: 스마트폰 내장 저장소·SSD

---

### 23. MCP / eMCP (Multi-Chip Package) 〔패키징 제품 형태〕

- **정의**: 여러 개의 메모리 칩을 하나의 패키지 안에 적층한 제품
- **종류**: NAND MCP(NAND만 적층) / eMCP(NAND+LPDDR 통합) / eUFS(eMCP 후속)
- **적층 방식**: 속도보다 원가가 중요 → Wire Bonding 방식 주류 (HBM의 TSV와 대비)
- **시장 구조**: 프리미엄폰=NAND+LPDDR 별도 탑재 / 중저가폰=eMCP 통합
- **관련 기업**: 삼성전자·SK하이닉스 양분

---

### 24. MES (Manufacturing Execution System, 제조실행시스템) 〔제조 현장 관리 시스템〕

- **정의**: 공장 현장에서 실시간으로 생산 공정을 모니터링·제어·기록하는 시스템
- **관리 범위**: 설비 가동 상태, 작업 지시, 품질 데이터, 수율, 작업자 현황
- **ERP와의 차이**: ERP="무엇을 얼마나 만들어라"(계획) / MES="지금 현장에서 실제 벌어지는 일"(실행)
- **반도체에서 중요성**: 수백 단계 공정, 수천 대 설비 → MES 없이 수율 관리 불가
- **IR 맥락**: 스마트팩토리 전환 핵심 소프트웨어

---

### 25. ERP (Enterprise Resource Planning, 전사적 자원관리) 〔경영 전반 통합 관리 시스템〕

- **정의**: 기업의 재무·인사·구매·물류·생산계획 등 경영 전반을 통합 관리하는 시스템
- **MES와의 관계**: ERP(상위·경영층, 계획) ↕ 데이터 연동 ↕ MES(하위·현장, 실시간)
- **주요 공급사**: SAP(글로벌 1위), Oracle, 더존비즈온(국내)
- **IR 맥락**: 클라우드 ERP 전환(SAP S/4HANA 등)이 현재 주요 트렌드

---

### 26. MLCC (Multi-Layer Ceramic Capacitor, 적층 세라믹 커패시터) 〔전자 수동부품〕

- **정의**: 전자회로에서 전기를 일시 저장했다가 순간 공급해 전압을 안정화하는 부품
- **배터리와 차이**: 배터리=대용량·느린 충방전 / 커패시터=소용량·초고속 충방전
- **구조**: 세라믹 유전체층 + 금속 전극층을 수백~수천 층 교대 적층
- **크기**: 최소 0201 규격(0.2×0.1mm). 스마트폰 약 1,000개, 전기차 약 15,000개 탑재
- **수요 드라이버**: 5G(탑재량 2배), 전기차(10배), AI 서버 고전력 안정화
- **전압 변동 원인**: 부하 변동(전류 급변), 스위칭 노이즈(트랜지스터 on/off), 배선 인덕턴스, 외부 EMI
- **방치 시 문제**: 오작동(0/1 오인식), 소자 손상(게이트 산화막 파괴), 연쇄 오류, EMI 규제 위반
- **관련 기업**: 무라타(글로벌 1위·일본), TDK(2위), 삼성전기(3위·국내)

---

### 27. 하이브리드 본딩 (Hybrid Bonding) 〔차세대 패키징 연결 기술〕

- **정의**: 솔더볼 없이 구리 전극 + 산화막을 직접 맞붙여 칩을 연결하는 차세대 패키징 기술
- **패키징 기술 진화**: Wire Bonding → BGA(솔더볼) → TSV → 하이브리드 본딩(직접 접합)
- **"하이브리드" 이유**: 금속(Cu) 접합 + 절연체(SiO₂) 접합을 동시에 수행
- **BGA 대비 장점**: 피치 ~100μm → ~10μm 이하, 연결 밀도 10~100배↑
- **HBM 연결**: 현재 HBM3E=TSV+MR-MUF / 차세대 HBM4+=TSV+하이브리드 본딩 예상
- **관련 기업**: 한미반도체(장비), 이오테크닉스(레이저 어닐링)

---

### 28. PEMFC (Proton Exchange Membrane Fuel Cell) 〔수소 연료전지 - 저온형〕

- **정의**: 고분자(폴리머) 막을 전해질로 사용하는 저온 작동 연료전지
- **원리**: 수소(H₂)+산소(O₂) → 전기+물. H⁺(양성자)만 막 통과
- **핵심 특성**: 작동온도 60~80℃, 빠른 시동
- **핵심 부품**: MEA = PEM막(나피온·불소폴리머) + 백금 촉매 + GDL
- **주요 용도**: 수소차(현대 넥쏘·도요타 미라이)
- **IR 리스크**: 백금 촉매 원가(부품비 40%+), PEM막 나피온 독점(듀폰·미국)
- **관련 기업**: 현대차, 코오롱인더스트리

---

### 29. SOFC (Solid Oxide Fuel Cell) 〔수소 연료전지 - 고온형〕

- **정의**: 세라믹 산화물을 전해질로 사용하는 초고온 작동 연료전지
- **원리**: O²⁻(산소이온)이 세라믹 전해질 통과
- **핵심 특성**: 작동온도 600~1,000℃, 효율 60~85%+
- **연료**: 수소·천연가스·암모니아 등 다양 (내부 개질 가능)
- **장점**: 백금 불필요(니켈 촉매), 배출열 재활용
- **주요 용도**: 대형 발전소, AI 데이터센터 전원
- **관련 기업**: Bloom Energy(미국), 두산퓨얼셀

---

### 30. 다이오드 (Diode) 〔반도체 기본 소자〕

- **정의**: 전기를 한 방향으로만 흐르게 하는 반도체 소자. 전기의 체크밸브
- **트랜지스터와 차이**: 다이오드=2단자·수동 / 트랜지스터=3단자·능동
- **주요 종류**: 일반(정류), LED, 제너(전압안정화), 쇼트키(고속스위칭), 레이저 다이오드(광통신 핵심)

---

### 31. 실리콘 포토닉스 (Silicon Photonics) 〔광 회로 기술〕

- **정의**: 기존 CMOS 반도체 공정으로 빛을 처리하는 회로를 실리콘 칩 위에 구현하는 기술
- **기존 광부품 대비 장점**: 특수 화합물 반도체(InP·GaAs) → 실리콘 대체 → 저가·대량생산
- **핵심 구성**: 광 도파로 + 변조기(전기→광) + 포토디텍터(광→전기)
- **관련 기업**: Intel, Cisco, 오이솔루션(국내), TSEM(파운드리)

---

### 32. CPO (Co-Packaged Optics) 〔AI 데이터센터 아키텍처〕

- **정의**: 광통신 부품(광엔진)을 스위치 칩과 같은 패키지 안에 통합하는 기술
- **핵심 개선**: 전기→광 변환 거리를 수미터 → 수mm로 단축 → 전력↓, 속도↑
- **구성**: 스위치 ASIC + 광엔진 + 실리콘 포토닉스 칩
- **관련 기업**: AVGO(스위치ASIC·CPO 핵심수혜), Intel(광엔진)

---

### 33. SCADA (Supervisory Control and Data Acquisition) 〔산업 인프라 원격 감시·제어〕

- **정의**: 대규모 산업 설비·인프라를 원격으로 감시하고 제어하는 시스템
- **3대 시스템 계층**: ERP(경영·계획) ↔ MES(현장·공정) ↔ SCADA(인프라·유틸리티)
- **주요 적용처**: 전력망·수도·가스·반도체팹 유틸리티
- **보안 이슈**: 스턱스넷(2010) 대표 사례 → OT 보안 시장 성장
- **관련 기업**: Siemens, LS ELECTRIC(국내)

---

### 34. InP (인화인듐) 웨이퍼 〔광통신 핵심 소재〕

- **정의**: 인듐(In)과 인(P)의 화합물 반도체 웨이퍼. 빛 발생·처리에 최적화된 소재
- **역할**: 레이저 다이오드 제조의 핵심 원재료
- **관련 기업**: AXTI(AXT Inc.)

---

### 35. 광 트랜시버 모듈 (Optical Transceiver Module) 〔광통신 완성 부품〕

- **정의**: 레이저·SiPh 칩·DSP 등을 조립해 전기↔광 변환을 수행하는 완성 모듈
- **규격 진화**: 100G → 400G → 800G → 1.6T
- **CPO와의 관계**: CPO 전환 시 외장 모듈 수요 감소 리스크
- **관련 기업**: AAOI(Applied Optoelectronics)

---

### 36. 광엔진 (Optical Engine) 〔CPO 핵심 부품〕

- **정의**: 레이저·변조기·포토디텍터·드라이버 IC를 단일 모듈로 통합
- **CPO에서의 역할**: 스위치 칩 옆에 붙어 전기↔광 변환 담당
- **관련 기업**: POET Technologies(InP+실리콘 단일기판 통합 기술, 상용화 초기)

---

### 37. 기판 (Substrate, PCB/Package Substrate) 〔반도체 패키징 핵심 부품〕

- **정의**: 반도체 칩과 외부 회로를 전기적으로 연결해주는 다층 배선판
- **역할**: 전기 연결, 기계적 지지, 열 방출
- **종류**: PCB(일반) / Package Substrate(반도체용) → BT기판(일반) / ABF기판(고성능) / FC-BGA
- **관련 기업**: 삼성전기·LG이노텍(국내), Ibiden·Shinko(일본)

### 38. ABF 기판 (Ajinomoto Build-up Film Substrate) 〔고성능 반도체용 패키지 기판〕

- **공급 구조**: ABF 필름은 아지노모토(일본) 사실상 독점
- **관련 기업**: Ibiden·Shinko(일본), 삼성전기·LG이노텍(국내)

### 39. 레이저 어닐링 (Laser Annealing) 〔반도체 열처리 공정〕

### 42. 반도체 IP (Semiconductor IP) 〔재사용 가능한 회로 설계 블록〕

- **정의**: 반도체 설계에 쓰이는 재사용 가능한 회로 설계 블록. 칩 설계의 "레고 블록"
- **필요 이유**: 처음부터 전부 설계하면 시간·비용 막대 → 검증된 IP 라이선스로 개발기간 단축
- **종류**: 코어 IP(CPU·GPU 코어), 인터페이스 IP(USB·PCIe·DDR), 아날로그 IP(전력관리)
- **대표 사례**: ARM(CPU 코어 설계, 직접 제조 안함) → 삼성·퀄컴·애플이 라이선스 받아 자체 칩 설계
- **비즈니스 모델**: 칩 판매량당 로열티 수익 → 반도체 업황 무관하게 안정적. 높은 진입장벽
- **관련 기업**: ARM(코어), Synopsys·Cadence(설계IP·EDA). AI시대 NPU IP 신규 부상

### 43. 화합물 반도체 (Compound Semiconductor) 〔다원소 화합 반도체 소재〕

- **정의**: 실리콘 같은 단일 원소가 아닌 두 가지 이상의 원소를 화합해 만든 반도체 소재
- **실리콘과 차이**: Si=단일원소·범용·저가 / 화합물=특수물성·고성능·고가
- **주요 종류**

| 종류 | 특성 | 주요 용도 |
|------|------|----------|
| GaN(질화갈륨) | 고전압·고주파 | 전력반도체, 5G RF |
| SiC(탄화규소) | 고전압·고온내성 | 전기차 인버터 |
| GaAs(갈륨비소) | 고주파 | RF칩, 위성통신 |
| InP(인화인듐) | 광특성 우수 | 레이저·광통신 |

- **실리콘 포토닉스와 구분**: 화합물반도체(InP 등)=빛을 만드는 소재(레이저) / 실리콘포토닉스=빛을 처리·전달하는 회로기술. 실제 광통신은 둘의 결합
- **IR 맥락**: 전기차 인버터(SiC) 채택 시 전력손실↓·주행거리↑. 현대차·LG이노텍 SiC 투자

### 44. 직류 (DC, Direct Current) / 교류 (AC, Alternating Current) 〔전기의 형태〕

- **직류(DC)**: 전류가 한 방향으로만 일정하게 흐르는 전기. 예: 배터리, 태양광, USB
- **교류(AC)**: 전류의 방향·크기가 주기적으로 바뀌는 전기. 예: 가정용 전기(220V, 60Hz)
- **AC로 송전하는 이유**: 변압기로 전압 변환이 쉬워 장거리 송전 손실↓
- **변환 필요성**: 발전소(AC송전) → 가정(AC공급) → 전자기기 내부(DC 작동) → **전력반도체가 AC↔DC 변환 담당**

---

### 45. 전력반도체 (Power Semiconductor) 〔전기 에너지 변환·제어 반도체〕

- **정의**: 전기 에너지를 변환·제어·분배하는 반도체. 일반 반도체(정보처리)와 달리 큰 전류·전압을 다룸
- **핵심 역할**: AC→DC 변환(정류), DC→AC 변환(인버터), 전압 승압/감압, 전류 스위칭
- **주요 소자**: 다이오드(정류), MOSFET(저전압·고속), IGBT(고전압·고전류), GaN·SiC(화합물 전력반도체)
- **전기차 핵심 활용**
```
[배터리]DC → [인버터: IGBT/SiC]DC→AC 변환 → [모터]AC 구동
```
SiC 채택 시 변환 손실↓ → 동일 배터리로 주행거리↑
- **아날로그 반도체와의 관계**: 전력반도체 ⊂ 아날로그 반도체 (포함관계)
- **관련 기업**: Infineon(글로벌 강자), 현대차·LG이노텍(SiC 투자)

---

### 46. 아날로그 반도체 (Analog Semiconductor) 〔연속신호 처리 반도체〕

- **정의**: 연속적인 신호(전압·전류 크기 변화)를 처리하는 반도체. 디지털(0과1)과 대비
- **디지털과 차이**: 디지털=0/1만 처리(CPU·메모리) / 아날로그=연속값 처리(실제 세계 신호 대응)
- **신호 변환**: ADC(아날로그→디지털) ↔ DAC(디지털→아날로그)
- **분류**: 전력관리IC(PMIC), ADC/DAC, 증폭기, 전력반도체(하위 포함)
- **산업 특징**: 미세공정 경쟁 덜함(28nm 이상도 충분), 설계 노하우·신뢰성이 핵심 진입장벽, 사이클 변동성 낮음(자동차·산업용 장기계약)
- **관련 기업**: Texas Instruments·Analog Devices(글로벌), LX세미콘·매그나칩·DB하이텍(국내)

---

*마지막 업데이트: AC·DC·전력반도체·아날로그반도체 세션 15*

### 47. MOSFET (Metal-Oxide-Semiconductor Field-Effect Transistor) 〔전압 제어 트랜지스터〕

- **정의**: 전압으로 전류를 제어하는 트랜지스터. 현대 반도체의 90%+ 가 이 구조
- **구조**: 게이트(전압 입력)에 전압을 걸면 소스-드레인 채널이 열려 전류 흐름
- **BJT와 차이**: BJT=전류로 전류제어(전력소모 큼) / MOSFET=전압으로 전류제어(저전력·고속)
- **용도 구분**: 로직용(CPU·메모리 내부 디지털 연산) vs 파워 MOSFET(전력반도체용, 저전압·고속 스위칭)
- **IGBT와 비교**: 파워MOSFET=저전압·고속 / IGBT=고전압·고전류 (전기차 메인 인버터는 IGBT·SiC 선호)
- **수요처**: 전기차 보조전원, 충전기·어댑터, 데이터센터 전원관리
- **관련 기업**: Infineon·ON Semiconductor(글로벌), KEC·매그나칩(국내)

---

### 48. CCL (Copper Clad Laminate, 동박적층판) 〔기판 제조 원재료〕

- **정의**: 절연판 양쪽에 구리(Cu)박을 붙인 판. PCB·패키지 기판을 만드는 핵심 원재료
- **구조**: 구리박 - 절연수지(에폭시+유리섬유) - 구리박
- **제조 흐름 위치**: **CCL(소재)** → PCB/기판 회사가 가공(회로패턴 식각·적층) → 완제품 기판
- **분류**

| CCL 종류 | 용도 |
|---------|------|
| 일반 FR-4 CCL | 일반 전자기기 PCB |
| BT CCL | 반도체 패키지 기판(저가) |
| ABF | 고성능 반도체 패키지(아지노모토 필름) |

- **관련 기업**: 두산전자(패키지기판 CCL 강자), 율촌화학(국내), Panasonic·Isola(해외)

---

### 49. MLB (Multi-Layer Board, 다층기판) 〔다층 적층 완제품 기판〕

- **정의**: 배선층을 여러 층으로 쌓은 PCB·패키지 기판
- **다층화 이유**: 단층으로는 복잡한 회로 구현 불가 → 층을 쌓아 같은 면적에 더 많은 회로 수용
- **CCL과의 관계**: CCL(원재료) → 가공 → MLB(다층 적층 완제품)
- **층수와 성능**: 층수↑ → 신호선 수용 능력↑ → 고성능 칩(GPU·AI칩) 대응 가능, 단 단가도 동반 상승
- **IR 맥락**: AI 서버·HBM 패키징 고도화로 요구 층수 증가(8~12층→16층+) → 기판 업체 ASP 상승 요인
- **관련 기업**: 삼성전기, LG이노텍, 대덕전자, 심텍

---

*마지막 업데이트: MOSFET·CCL·MLB 세션 16*

### 50. 인버터 (Inverter) 〔DC→AC 전력변환 장치〕

- **정의**: DC를 AC로 변환하는 장치. 전력반도체(IGBT·SiC)가 변환을 실제 수행
- **전기차 역할**: 배터리(DC) → 인버터(DC→AC + 주파수 조절) → 모터(AC 구동, 회전속도 제어)
- **구성**: IGBT/SiC 모듈 + 제어회로 + 냉각구조
- **IR 맥락**: 인버터 효율=주행거리 직결. SiC 인버터 채택 확대가 핵심 트렌드(테슬라 선도)

---

### 51. 컨버터 (Converter) 〔전압 크기 변환 장치〕

- **정의**: 전압의 크기를 변환하는 장치 (인버터=AC↔DC 변환과 구분)
- **종류**: DC-DC 컨버터(400V→12V 등), AC-DC 컨버터(충전기 내부)
- **전기차 역할**: 고전압 배터리(400V/800V) → DC-DC 컨버터 → 저전압(12V) → 헤드라이트·인포테인먼트 등 보조 전원
- **인버터와 구분**: 인버터=메인 동력(고전력, IGBT/SiC) / 컨버터=보조전원(상대적 저전력, MOSFET)

### 52. 버스바 (Busbar) 〔대전류 분배 도체 — EV·ESS·변압기 공통〕

- **정의**: 대전류를 한 지점에서 여러 지점으로 분배하는 금속 도체 구조물. EV 배터리팩, ESS, 변압기, 데이터센터 UPS 등 대전류 시스템 어디에나 들어가는 "전기 인터체인지".
- **일반 전선과 차이**: 수백~수천 A 대전류 → 일반 전선은 저항·발열 과다 → 단면적이 크고 단단한 금속판(구리·알루미늄) 사용

#### 52-A. 세 가지 방식 비교

| 항목 | Solid 버스바 | Flexible 버스바 | 무용접 일체형 버스바 |
|---|---|---|---|
| **구조** | 단일 구리/알루미늄 덩어리 가공 | 박판(foil) 여러 겹 적층 | 단일 연속 도체를 성형(굽힘·3D 가공) |
| **접촉저항** | 없음(단일체) | **있음** — 층간 계면 다수 | **없음** — 계면 자체 미존재 |
| **진동 내성** | 낮음 — 강성 높아 체결부 피로 | 중간 — 층간 마이크로슬립 발생 | 높음 — 연속체가 탄성 흡수 |
| **열팽창 대응** | 취약 — 마운트 포인트에 스트레스 집중 | 중간 | 양호 — 굽힘 형상이 신축 흡수 |
| **제조 공정** | 단순 (절삭 가공) | 복잡 (적층·브레이징·단자 부착) | 단순 (성형 후 검사) |
| **비용** | 기준 | 기준+브레이징 비용 | 20~30% 절감 주장 |
| **리드타임** | 긴 편 (금형 변경 필요) | 김 | 20~40% 단축 주장 |
| **단자 일체화** | 별도 가공 필요 | 어려움 | 일체형 성형 가능 |

#### 52-B. 무용접 일체형의 핵심 공학 이점

1. **접촉저항 Zero → 발열 근원 제거**
   - 발열량 Q = I²R. 대전류(500A)에서 접촉저항 1mΩ = 250W 손실 → 일체형은 이 손실이 구조적으로 불가
   - 장기 운용 중 산화에 의한 저항 증가 메커니즘도 차단

2. **진동·충격 면역** — 단일 도체 내부에 슬립 발생 면이 없으므로 마이크로 아킹·아크 마모 없음

3. **제조 공정 단순화** — 용접·브레이징 공정 제거 → 가스(SF₆ 등) 사용 없음 → 탄소저감 + 원가절감

#### 52-C. IR 맥락 (투자 관점)

- **타깃 시장**: 버스바 글로벌 TAM 38조원(2026, CAGR 9%) — EV 배터리팩, ESS, 변압기 모두 포함
- **Why now**: EV·ESS 폭증 + 화재·진동 안전 규제 강화 → 성능 우위 제품 수요 구조적 팽창
- **투자 리스크**: "접촉저항 Zero", "20~30% 원가절감" 수치는 회사 자체 주장 — KOLAS 실측값·고객사 수령 샘플 제3자 검증이 투자 판단의 핵심

### 56. 벡터 DB (Vector Database) 〔AI·LLM 시대 핵심 검색 인프라〕

- **정의**: 데이터를 숫자 벡터(좌표)로 변환 저장해 "의미가 비슷한" 데이터를 검색하는 DB
- **기존 DB와 차이**: 일반DB="정확히 일치하는가"(Exact Match) / 벡터DB="좌표상 가장 가까운가"(Nearest Neighbor Search)
- **원리**: 임베딩 모델이 텍스트·이미지를 숫자 배열로 변환 → 의미가 비슷할수록 좌표 공간에서 가까운 위치
- **RAG와의 관계**
```
사용자 질문 → 벡터DB에서 관련 문서 의미기반 검색
→ 검색된 문서를 LLM에 전달 → LLM이 참고해 답변 생성
→ LLM이 모르는 회사 내부 정보도 정확히 답변 가능
```
- **대표 솔루션**: Pinecone·Weaviate·Milvus·Chroma(전용), pgvector(PostgreSQL 확장)
- **AI 인프라 스택 위치**: 하드웨어(GPU·HBM·CPO) ↔ 소프트웨어(LLM·벡터DB·RAG프레임워크)
- **IR 맥락**: 기업 자체 데이터로 LLM 활용(RAG) 수요 증가 → 벡터DB 인프라 투자 활발. 클라우드 기업(AWS·MS)도 내재화 경쟁

### 57. 터빈 (Turbine) 〔에너지·발전 핵심 기계〕

- **정의**: 유체(증기·가스·바람·물)의 운동에너지를 회전력(기계에너지)으로 변환하는 장치
- **작동 원리**: 유체가 날개(블레이드)를 밀어 축을 회전 → 발전기 구동
- **주요 종류**

| 종류 | 작동 유체 | 용도 |
|------|----------|------|
| 증기터빈 | 수증기 | 석탄·원자력·SOFC 복합발전 |

| 수력터빈 | 물 | 댐 발전 |

- **관련 기업**: GE Vernova(미국), Siemens Energy(독일), 두산에너빌리티(국내·가스터빈 국산화)

### 58. 네트워크 스위치 (Network Switch) 〔AI 클러스터 통신 핵심 장비〕

- **정의**: 데이터 패킷을 받아 목적지 MAC 주소를 보고 해당 포트로 전달하는 네트워크 장비
- **허브와 차이**: 허브=모든 포트에 무차별 전송 / 스위치=목적지만 선택 전송 → 효율적
- **AI 인프라에서의 역할**
```
GPU #1 ─┐
GPU #2 ─┤
GPU #3 ─┼── [네트워크 스위치] ── 다른 서버 랙
GPU #4 ─┤
...     ─┘
수천 개 GPU가 All-to-All 통신 → 초저지연·초고대역폭 스위치 필수
```
- **CPO와의 연결**: 스위치 칩(ASIC) + 광엔진을 같은 패키지에 → CPO. 스위치가 CPO 수요의 근원
- **규격 진화**: 12.8Tbps → 51.2Tbps → 102.4Tbps (AI 클러스터 규모 확대에 따라)
- **IR 맥락**: GPU 클러스터 규모↑ → 스위치 대역폭 요구 폭발적 증가. Broadcom이 스위치 ASIC 시장 사실상 독점(Tomahawk·Jericho 시리즈) → AI 인프라 투자의 숨은 수혜자
- **관련 기업**: Broadcom(스위치 ASIC 독점적 지위), Arista Networks·Cisco(완성 장비), 드래곤플라이(국내)

### 59. 액추에이터 (Actuator) 〔로보틱스·자동화의 "근육"〕

- **정의**: 전기·유압·공압 에너지 신호를 받아 실제 물리적 움직임(회전·직선운동)을 만드는 장치
- **로봇 구성에서의 위치**
```
센서(눈) → 컨트롤러(뇌) → 액추에이터(근육·팔다리) → 물리적 동작
```
- **인버터와의 연결**: 인버터(DC→AC 변환) → 모터 구동 → 모터=회전형 액추에이터
- **주요 종류**

| 종류 | 에너지원 | 특징 | 용도 |
|------|---------|------|------|
| 전동 액추에이터 | 전기(모터) | 정밀제어 용이 | 협동로봇, 반도체 장비 |
| 유압 액추에이터 | 오일 압력 | 고출력 | 중장비, 산업로봇 |
| 공압 액추에이터 | 압축공기 | 빠른 동작 | 공장 자동화 |

- **IR 맥락**: 휴머노이드 로봇 1대에 액추에이터 수십 개 탑재. 정밀 감속기(하모닉드라이브) 일체형 액추에이터가 로봇 원가의 30~40% 차지 → 로봇 투자 시 액추에이터·감속기 밸류체인이 핵심 체크포인트
- **관련 기업**: Harmonic Drive(일본·감속기 독점적 지위), Maxon(스위스), 레인보우로보틱스·뉴로메카(국내)

### 60. API (Application Programming Interface) 〔소프트웨어 간 통신 규약〕

- **정의**: 소프트웨어끼리 기능을 주고받기 위한 표준화된 요청·응답 규약
- **비유**: 레스토랑 주문서. 손님(앱)이 정해진 형식으로 주문(요청)하면 주방(서버)이 요리(결과)를 돌려줌. 주방 내부 레시피는 몰라도 됨

```
앱(클라이언트) → [API 요청: "날씨 알려줘"] → 서버
앱(클라이언트) ← [API 응답: "서울 28℃"]  ← 서버
```
- **AI API의 부상**: LLM 기능을 API로 외부 제공 → 개발자가 Claude·GPT 등을 자사 서비스에 바로 연동
- **비즈니스 모델 관점**

| 모델 | 특징 | 예시 |
|------|------|------|
| 사용량 기반(Usage-based) | 호출 횟수·토큰 수 과금 | Claude API, OpenAI API |
| 구독형(Subscription) | 월정액 API 접근권 | Twilio, Stripe |
| 수익공유(Revenue share) | 파트너 매출 연동 | 카카오페이 API |

- **IR 맥락**: API 모델 = 사용량 기반 반복수익(Usage-based ARR). 플랫폼 락인 강력(전환비용 높음) → 높은 NRR(130%+). AI API 시장은 기업용 AI 도입의 관문 → 사용량 복리 성장 구조. 앤트로픽·OpenAI의 기업가치 근거
- **관련 기업**: Anthropic(Claude API), OpenAI(GPT API), Twilio·Stripe(비AI API 레퍼런스)

### 61. Adapter (AI 모델 어댑터) 〔Foundation Model 전문화 기법〕

- **정의**: 대형 AI 모델(Foundation Model/Core)의 가중치는 동결한 채, 소규모 추가 레이어만 도메인 데이터로 학습해 특정 분야에 특화시키는 기법
- **Full Fine-tuning과 비교**

```
[Full Fine-tuning]
Core 전체 재학습 → 비용·시간 막대, 도메인마다 별도 모델

[Adapter 방식]
Core 동결(Freeze) → [소형 Adapter Layer만 학습] → 빠르고 저렴
```

- **"One Core · Many Domains" 구조**

| Adapter | 역할 | 특화 데이터 |
|---------|------|------------|
| Safety Adapter | 위험 감지·제어 | 사고 시나리오, 안전 규정 |
| Manufacturing Adapter | 공정 최적화·이상감지 | 반도체·배터리 공정 데이터 |
| Vision/Quality Adapter | 외관 검사·불량 판별 | 양품·불량 이미지 |
| Action Adapter | 로봇 동작 제어 | 모방학습 데이터 |

- **기술적 한계**
  - Core 표현력 상한: Core가 학습한 적 없는 도메인은 Adapter로 메우기 어려움
  - Adapter 간 충돌(Interference): 여러 Adapter 동시 활성화 시 신호 충돌 가능
  - 실시간 제어 지연: Adapter 레이어 추가로 추론 시간 증가 → 수ms 반응 필요 환경에서 불리
  - 데이터 역설: 데이터가 충분히 많을 때는 Full Fine-tuning이 성능 우위
- **IR 맥락**: Adapter 구조의 설득력은 결국 **Core 품질**에 달림. 경쟁사도 "Adapter 구조 채택" 가능 → 차별화 근거는 Core 학습 데이터·아키텍처 우위. 플랫폼 확장성 스토리의 기술적 전제조건 검증이 핵심 DD 포인트
- **관련 기업**: Hugging Face(LoRA·PEFT 오픈소스 Adapter 생태계), Nenya(sVLA Core 기반 도메인 Adapter 구조)

### 62. 초임계 유체 (Supercritical Fluid, SCF) 〔반도체 세정·건조 핵심 소재〕

- **정의**: 온도와 압력이 임계점을 초과해 기체와 액체의 경계가 사라진 제3의 물질 상태
- **상변화 원리**

```
온도·압력 상승
액체 ──────────► 임계점(Critical Point) ──► 초임계 유체
                  (CO₂: 31℃, 74기압)         기체+액체 특성 동시 보유
```

- **핵심 물성 (초임계 CO₂ 기준)**

| 특성 | 값 | 의미 |
|------|-----|------|
| 표면장력 | **0** | 나노 패턴 내부 침투·건조 시 패턴 붕괴 없음 |
| 밀도 | 액체 수준 | 용해력 강함 (오염물 제거) |
| 점도 | 기체 수준 | 좁은 공간 빠른 침투 |
| 확산속도 | 기체 수준 | 균일한 처리 |

- **반도체 공정 적용**
  - **세정**: 2nm 이하 초미세 패턴 내 이물질 제거. 일반 액체 세정제는 표면장력으로 패턴 쓰러짐(Pattern Collapse) 발생 → 초임계 CO₂로 해결
  - **건조**: EUV 후 포토레지스트 현상 건조 공정. 기존 IPA 건조 대체
- **타 산업 응용**: 식품(무용매 카페인 추출), 2차전지 양극재 전구체 합성·건조, 의약품 추출
- **IR 맥락**: 반도체 미세화(2nm→1.4nm→이하)로 기존 공정 한계 도달 → 초임계 공정이 필수 대안으로 부상. TSMC·삼성 선단 공정 채택 가속 → 초임계 장비·CO₂ 정제 소재 기업 수혜. 국내 기업들 장비 국산화 시도 중
- **관련 기업**: 삼성전자·TSMC(수요처), 피에스케이·원익IPS(국내 장비), SFT(Supercritical Fluid Technologies, 해외)

### 63. 블루투스 (Bluetooth) 〔근거리 무선통신 - PAN〕

- **정의**: 2.4GHz 주파수 대역을 사용하는 근거리 무선통신 기술. 기기 간 직접 연결에 최적화
- **분류 3축 포지셔닝**: 거리=PAN(~10m) / 구조=P2P(1:1) 또는 Star / 트레이드오프=저전력 우선
- **핵심 구분**

| 종류 | 속도 | 전력 | 용도 |
|------|------|------|------|
| Classic BT | 빠름 | 높음 | 음악·파일 전송 |
| BLE (저전력) | 느림 | 매우 낮음 | IoT 센서·웨어러블 |

- **버전 진화**: BT 4.0(BLE 추가·IoT 혁신) → BT 5.0(거리 4배·속도 2배) → BT 5.4(브로드캐스트 개선·전자가격표)
- **Wi-Fi와 차이**: BT=기기 간 직접 연결 / Wi-Fi=AP 경유 인터넷 연결. BT가 전력·비용 유리
- **IR 맥락**: 무선이어폰·웨어러블·스마트홈 기기마다 BT 칩 필수 탑재. BLE는 IoT 기기 배터리 수명 혁신의 핵심 기술
- **관련 기업**: Qualcomm(BT 칩 강자), Nordic Semiconductor(BLE 전문), 삼성전기(모듈)

---

### 64. Wi-Fi 〔근거리 무선 인터넷 - LAN〕

- **정의**: AP(공유기)를 통해 기기를 인터넷에 연결하는 무선 LAN 기술. 2.4GHz / 5GHz / 6GHz 대역 사용
- **분류 3축 포지셔닝**: 거리=LAN(~100m) / 구조=Star(AP 중심) / 트레이드오프=속도 우선
- **세대별 진화**

| 규격 | 이름 | 최대속도 | 핵심 개선 |
|------|------|---------|----------|
| 802.11ac | Wi-Fi 5 | 3.5Gbps | 5GHz 주류화 |
| 802.11ax | Wi-Fi 6/6E | 9.6Gbps | 밀집환경 강화(OFDMA), 6GHz 추가 |
| 802.11be | Wi-Fi 7 | 46Gbps | 멀티링크(2개 대역 동시 사용) |

- **블루투스와 차이**: 속도·범위·전력소비 모두 Wi-Fi가 큼. Wi-Fi는 AP 없이 통신 불가(Star 구조의 취약점)
- **IR 맥락**: Wi-Fi 6E·7 전환 → AP 교체 사이클 + 칩 업그레이드 수요. Broadcom·MediaTek·Qualcomm이 Wi-Fi 칩 시장 과점
- **관련 기업**: Broadcom·Qualcomm·MediaTek(칩), Cisco·Aruba(기업용 AP)

---

### 65. UWB (Ultra-Wideband, 초광대역) 〔정밀 위치측정 무선통신〕

- **정의**: 넓은 주파수 대역(3.1~10.6GHz)에 극히 짧은 펄스를 쏴서 ±10cm 수준의 정밀 위치를 측정하는 기술
- **분류 3축 포지셔닝**: 거리=PAN(~수십m) / 구조=Star 또는 Mesh / 트레이드오프=정밀도 극대화
- **블루투스·GPS와 차이**

| 기술 | 위치 정확도 | 특징 |
|------|-----------|------|
| GPS | ±수m | 실외만 작동 |
| BLE | ±수m | 저전력이나 부정확 |
| UWB | **±10cm** | 실내 정밀 측위 가능 |

- **주요 용도**: 애플 AirTag(분실물 추적), 스마트카 디지털키(사람이 차 옆에 있을 때만 열림), 공장 내 자재·장비 위치추적
- **IR 맥락**: Apple U1 칩 탑재로 시장 견인. 자율주행·스마트팩토리·스마트홈에서 "정밀 위치"가 핵심 인프라로 부상
- **관련 기업**: Qorvo·NXP(UWB 칩 시장 양분), Apple(U1 칩 자체 설계)

---

### 66. LoRa (Long Range) 〔저전력 장거리 IoT 통신 - LPWAN〕

- **정의**: 극저전력·장거리 무선통신 기술. LPWAN(Low Power Wide Area Network)의 대표 규격
- **분류 3축 포지셔닝**: 거리=WAN(수km~수십km) / 구조=Star(게이트웨이 중심) / 트레이드오프=전력·거리 극대화, 속도 희생
- **포지셔닝**

```
빠름 ◄──────────────────────► 느림(0.3~50kbps)
     Wi-Fi    5G   NB-IoT   LoRa
짧음 ◄──────────────────────► 멀리(수십km)
전력↑                        전력↓(배터리 10년)
```

- **주요 용도**: 스마트 전기·수도 미터기, 농경지 토양 센서, 물류 컨테이너 추적, 가로등 원격제어
- **LoRaWAN**: LoRa 물리계층 위에 네트워크 프로토콜을 표준화한 규격. 통신사 운영망 vs 자체 사설망 두 갈래
- **IR 맥락**: Semtech이 LoRa 칩 원천기술 보유(독점적 지위). 스마트시티·농업IoT 인프라 구축 수요와 직결
- **관련 기업**: Semtech(LoRa 칩 원천), SKT·KT(LoRaWAN 국내 통신망 운영)

---

### 67. TSN (Time-Sensitive Networking) 〔실시간 보장 산업용 이더넷〕

- **정의**: 표준 이더넷에 마이크로초 단위 실시간 전송 보장 기능을 추가한 확장 규격(IEEE 802.1 시리즈)
- **일반 이더넷의 한계**: 패킷이 스위치 큐(대기열)에 쌓이면 지연이 불규칙하게 발생(지터, Jitter). "결국 도착"은 보장하지만 "언제 도착"은 보장 불가
- **TSN 해결 방식**: 네트워크 전체가 동일 시간 공유(IEEE 802.1AS) → 긴급 패킷 전용 시간 슬롯 예약 → 혼잡해도 슬롯 내 우선 통과

```
일반 이더넷: 모든 패킷 동일 차선 → 혼잡 시 지연 급증
TSN:        긴급 패킷 전용 슬롯 → 설계 범위 내 지연 보장
```

- **중요한 전제**: TSN은 "설계 범위 내에서" 보장. 예상 초과 트래픽이 몰리면(Over-subscription) 보장 실패 → Worst Case Traffic Analysis(최악 시나리오 설계)가 필수
- **미션크리티컬 환경 대응**: TSN + 트래픽 셰이핑 + 이중화 경로 + 하드와이어(물리 전선) 병행
- **IR 맥락**: 스마트팩토리 IT·OT 통합의 핵심 인프라. 장비 판매보다 설계 툴+장비+유지보수 통합 솔루션이 더 높은 마진 구조
- **관련 기업**: Cisco·Siemens·Phoenix Contact(장비), TTTech(TSN 설계 소프트웨어)

---

### 68. InfiniBand 〔데이터센터 서버 간 초고속 인터커넥트〕

- **정의**: 데이터센터·슈퍼컴퓨터에서 서버 간 초저지연·초고대역폭 통신을 위한 전용 네트워크 기술
- **일반 이더넷과 차이**

```
이더넷:     데이터 → CPU → OS → 메모리 (CPU 개입)
InfiniBand: 데이터 → 메모리 직접 전달, CPU 부하 없음 (RDMA)
지연시간:   이더넷 ~수십μs  vs  InfiniBand ~1μs
```

- **RDMA(Remote Direct Memory Access)**: CPU를 거치지 않고 원격 서버 메모리에 직접 접근. CPU 부하↓, 속도↑
- **AI 클러스터에서 역할**: GPU 수천 개가 학습 중 gradient(계산 결과)를 서로 교환. 이 통신이 느리면 GPU가 대기 상태로 낭비
- **이더넷(RoCE)과 경쟁**: RoCE=이더넷 위에 RDMA 구현. 범용성↑이지만 순수 InfiniBand 대비 성능·안정성 열위
- **IR 맥락**: NVIDIA가 Mellanox(InfiniBand 원천기업) 2019년 인수 → H100 클러스터의 컴퓨팅+네트워킹 동시 장악. AMD·Intel이 RoCE로 대안 제시 중
- **관련 기업**: NVIDIA(Mellanox 인수 후 독점), Marvell(이더넷 기반 대안 칩)

---

### 69. NVLink 〔NVIDIA GPU 간 전용 고속 인터커넥트〕

- **정의**: NVIDIA가 설계한 GPU 간 전용 고속 연결 기술. 같은 서버 내 GPU들을 초고속으로 연결
- **InfiniBand와 역할 구분**

```
[서버 내부]  GPU1 ─── NVLink ─── GPU2  ← NVLink 담당
             GPU3 ─── NVLink ─── GPU4
                  ↕ (서버 간)
[다른 서버]  InfiniBand 담당
```

- **PCIe 대비 성능**: PCIe ~64GB/s vs NVLink 4세대 ~900GB/s → 약 14배 대역폭
- **NVSwitch**: GPU 여러 개를 All-to-All로 연결하는 전용 스위치칩. H100 8개를 NVSwitch로 묶으면 하나의 거대 GPU처럼 동작(NVL8 구성)
- **락인 구조**: NVLink·NVSwitch는 NVIDIA 독점 생태계. AMD·Intel GPU와 호환 불가 → 한번 NVLink 클러스터 구축 시 전환비용 매우 높음
- **IR 맥락**: NVLink가 NVIDIA 생태계 락인의 핵심 기술적 근거. GPU 성능 경쟁에서 이기더라도 인터커넥트 호환성 문제로 교체 어려움 → NVIDIA 프리미엄 밸류에이션의 기반
- **관련 기업**: NVIDIA(독점), AMD(MI300X·자체 인터커넥트로 대응), Intel(Gaudi)

---

### 70. 이더넷 (Ethernet) 〔유선 네트워크 표준〕

- **정의**: 유선 네트워크의 사실상 표준 기술. 현재 거의 모든 인터넷 연결의 물리적 기반
- **설계 철학**: "데이터가 결국 도착하기만 하면 된다" → 정확성(도착 보장) O, 타이밍(언제 도착) X → TSN으로 보완
- **속도 진화**

```
1G → 10G → 25G → 100G → 400G → 800G → 1.6T
                    ↑            ↑
              일반 데이터센터   AI 클러스터 최전선
```

- **지연 불확실성 원인**: 스위치 큐(대기열) 혼잡 시 지연이 불규칙하게 발생(지터). 이메일·파일전송에는 무관하나 실시간 제어에는 치명적 → TSN 등장 배경
- **InfiniBand와 경쟁(RoCE)**: 이더넷 위에 RDMA 구현(RoCE). 범용·저가이나 순수 InfiniBand 대비 성능 열위. AI 클러스터 확산으로 격차 좁혀지는 중
- **IR 맥락**: 400G→800G 전환이 AI 클러스터 투자와 직결. 이더넷 칩·스위치 수요 폭증 → Broadcom·Marvell 핵심 수혜
- **관련 기업**: Broadcom·Marvell(이더넷 칩), Arista Networks·Cisco(스위치 장비)

---

### 71. 프로토콜 스택 / OSI 7계층 〔통신 계층 구조 이해 틀〕

- **정의**: 서로 다른 기기가 통신하는 규칙을 역할별로 7개 층으로 나눈 표준 모델
- **비유**: 국제 택배. 1계층=트럭·비행기(물리 이동) / 3계층=우편번호(경로) / 4계층=분실보상(신뢰성) / 7계층=편지 내용(앱)

```
7계층 (응용)     ← 사용자 앱 (카카오톡, 크롬)
6계층 (표현)     ← 데이터 형식·암호화(HTTPS)
5계층 (세션)     ← 연결 유지·관리
4계층 (전송)     ← TCP(신뢰) / UDP(속도) — 데이터 분할·오류검출
3계층 (네트워크) ← IP 주소, 경로 결정 (라우터)
2계층 (데이터링크)← MAC 주소, 같은 망 내 전달 (스위치)
1계층 (물리)     ← 실제 전기신호·빛 (케이블, Wi-Fi 전파)
```

- **VC가 알아야 하는 이유**: 어느 계층 솔루션인가에 따라 경쟁구도·진입장벽이 완전히 달라짐

| 계층 | 특성 | 투자 시사점 |
|------|------|-----------|
| 1~2계층 (물리·링크) | 하드웨어, 교체 어려움 | 높은 해자, 승자독식 |
| 3~4계층 (네트워크·전송) | 표준화, 차별화 어려움 | 범용 인프라 |
| 5~7계층 (응용) | 소프트웨어, 빠른 혁신 | 락인은 UX·데이터에서 발생 |

- **관련 기업**: 계층별로 완전히 다름. 1계층=광케이블(CORNING), 3계층=Cisco, 7계층=앱 기업

---

### 72. 토폴로지 (Network Topology) 〔네트워크 연결 구조〕

- **정의**: 네트워크에서 기기들이 어떤 구조로 연결되어 있는가를 나타내는 개념. VC 통신기술 분류 3축 중 ②번
- **주요 구조**

```
Star(중앙집중)   Mesh(분산)         P2P(1:1)       Ring(순환)
    [AP]        [A]─[B]─[C]       [기기]─[기기]   A─B─C─D─A
   / | \           |   |
기기 기기 기기   [D]─[E]─[F]
  Wi-Fi         Zigbee·Matter     블루투스Classic  산업이더넷
```

| 구조 | 장점 | 단점 | 대표 기술 |
|------|------|------|----------|
| Star | 관리 단순 | 허브 장애 시 전체 다운 | Wi-Fi, 기업 LAN |
| Mesh | 장애 내성, 자동 우회 | 복잡·비용↑ | Zigbee, Matter |
| P2P | 단순·빠른 연결 | 확장성 없음 | 블루투스 Classic |
| Ring | 예측 가능한 지연 | 한 곳 끊기면 취약 | 산업 이더넷 |

- **AI 클러스터의 Fat-Tree 토폴로지**: 스위치를 나무 형태 다층 구조로 쌓아 어느 GPU에서 어느 GPU로도 동일한 홉(경유 횟수)으로 도달 → 병목 없는 All-to-All 통신
- **IR 맥락**: 토폴로지 선택이 장비 수량을 결정. Mesh→각 노드에 칩 필요(칩 수요↑), Fat-Tree→스위치 수량 기하급수적 증가(네트워크 장비 기업 수혜)
- **관련 개념**: 프로토콜 스택(항목 71), 네트워크 스위치(항목 58), TSN(항목 67)

### 61. 지식 그래프 (Knowledge Graph, KG) 〔관계 기반 지식 표현 구조 — 벡터DB의 형제 개념〕

- **정의**: 개체(entity)를 노드로, 개체 간 관계(relationship)를 엣지로 표현한 그래프 구조의 데이터 모델. "무엇이 무엇과 어떻게 연결되어 있는가"를 컴퓨터가 탐색 가능한 형태로 저장
- **핵심 단위**: 트리플(Triple, 주어-술어-목적어) — 예: [삼성전자]--(생산한다)-->[HBM]--(탑재된다)-->[H100 GPU]
- **관계형 DB와 차이**: 관계형 DB=사전에 설계된 테이블 JOIN만 가능 / KG=관계 자체가 데이터라 다단계(multi-hop) 질의에 유연
- **벡터DB와 관계(형제 개념, 역할 다름)**: 벡터DB=의미 유사도 기반 퍼지 검색 / KG=정확한 관계를 따라가는 정밀 추론. 최근 GraphRAG로 결합 추세(벡터DB로 관련문서 검색 + KG로 개체 간 관계 정밀 추론)
- **온톨로지와 관계**: 온톨로지=KG가 따라야 할 스키마·규칙(설계도) / KG=그 규칙으로 채워진 실제 데이터(건물). "Company는 Product를 produces할 수 있다"가 온톨로지, 그 관계로 실제 채워진 그래프가 KG
- **IR 맥락**: 팔란티어 Foundry Ontology·서버키트 BeSir 등 엔터프라이즈 AI 스타트업 다수가 "KG 기반"을 핵심기술로 내세우나, 이는 이제 표준적 주장이라 차별화 근거가 되지 못함. 실사 핵심은 ①그래프 채우는 과정의 자동화 정도, ②여러 부서 데이터 통합 시 개체 중복·의미 충돌 해소 방식, ③규모 확장 시 성능 유지 여부
- **관련 기업**: 솔트룩스·마인즈랩(국내), Neo4j(그래프DB 1위), Palantir(Foundry Ontology), Amazon Neptune·Stardog(해외)
